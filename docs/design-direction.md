# Personal Brand Design Direction

## Core Idea

Kun is building and publishing systems that give one person the leverage of an enterprise.

Everything should feel like it comes from an **architect of individual leverage**.

---

## Visual System

### Typography Roles

- **Serif (Thesis):** Headlines, big ideas
- **Sans (Explanation):** Body text
- **Mono (Proof):** Code, commands, benchmarks

Never mix roles arbitrarily.

---

### Color

- Background: off-white / bone
- Text: near-black / graphite
- Accent: one signal color (e.g. deep green)

Use accent **only for meaning** (links, highlights, annotations).

---

### Layout

- Strong margins, lots of whitespace
- Thin dividers
- Grid-aligned sections
- No decorative elements

Design should feel like:

> editorial × system console × field manual

---

## Signature Element (MANDATORY)

### Inline Annotation Title

Every post must include a bracketed concept in the title.

Examples:

- One person should operate like an [enterprise]
- The future of work is [agent-native]
- Stop using tools. Start building a [system]

Rules:

- Only bracket the **core concept**
- Use accent color for the bracketed word
- Keep titles short and declarative

This is the primary brand identifier.

---

## Content Structure

### Standard Essay Format

1. **Claim** — one sharp sentence
2. **Broken Default** — what’s wrong today
3. **New Model** — name the concept
4. **Mechanism** — how it works
5. **Proof** — example, diagram, or code
6. **Implication** — what becomes possible

---

### Section Labels (Optional but Recommended)

Use small uppercase labels:

- PREMISE
- BROKEN DEFAULT
- NEW MODEL
- MECHANISM
- PROOF
- IMPLICATION

---

## Visual Artifacts (Required)

Every piece should include at least one:

- Diagram (before vs after)
- Workflow map
- Terminal / code block
- Benchmark or table

Avoid decorative illustrations.
Prefer **thinking artifacts**.

---

## Writing Style

- Precise, not performative
- Opinionated, but justified
- Systems-level thinking
- Calm tone, no hype

Avoid:

- Buzzwords
- Over-explaining
- Marketing language

---

## Repetition System

Consistency builds recognition:

- Same title pattern
- Same structure
- Same typography roles
- Same visual artifacts

Every post should feel like part of a **larger system**, not a standalone piece.
